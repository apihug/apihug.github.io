## 发生了什么

这个版本阶段引入了 ApiHug 早期的重要结构调整，后续很多版本都建立在这次迁移之上。

当时最重要的两项要求是：

1. 升级到新的 SDK / 插件线
2. 把工程对齐到 Spring Boot `3.2+`

## 升级步骤

### 项目版本

更新 `{PROJECT}/gradle/libs.versions.toml`：

```toml
apihug = "0.6.0-RELEASE"
```

### IntelliJ 插件

安装或升级 ApiHug IntelliJ 插件到 `0.2.0+`。

### Spring 兼容性

确认项目使用 Spring Boot `3.2+`，并检查第三方依赖是否兼容，尤其是 Spring Cloud 与数据访问相关组件。

## Proto 语法迁移

这个版本阶段涉及两项较早的语法变化：

1. `google.protobuf.BoolValue` 逐步迁移到 `hope.common.BoolType`
2. `authorization_struct` 改名为 `rbac`

布尔字段示例：

```proto
empty: false;
```

授权字段示例：

```proto
rbac: {
  authorities: "PLATFORM_MEMBER_OPERATE";
  combinator: OR;
  predefined_role_checker: PLATFORM_MANAGER
}
```

## 结果

完成这次迁移后，项目就可以进入新的 SDK / 插件版本线，并继续跟随后续里程碑，而不再停留在 0.6 之前的兼容分支。
