## 它是什么

共同语言，指的是团队拥有一套共享的表达方式，来描述领域、API 结构以及预期行为。

如果没有共同语言，同一个意图会在不同角色之间被不断重新翻译：

- 产品描述业务含义
- 架构师描述结构
- 开发描述实现
- 测试描述场景

每多一次翻译，就多一层损耗。

## 为什么重要

软件真正困难的地方，往往不是语法，而是对领域理解是否一致。

因此，在 API 工作里，共同语言尤其关键。只要命名、结构和语义表达不一致，项目就会持续为以下问题付出代价：

- 误解
- 评审反复
- 返工
- 文档漂移
- 集成错误

## 为什么需要领域化语言

面向领域的语言，可以缩短以下几类对象之间的距离：

- 领域专家
- 技术专家
- 可执行系统与测试

重点不是构建一套抽象理论，而是给团队一套可以讨论、版本化、评审并转化成真实产物的表达载体。

## ApiHug 如何应用它

ApiHug 通过共享契约语言，避免 API 定义被困在：

- 某一种编程语言里
- 某一种文档格式里
- 某一个角色的私人理解里

这份共享语言，正是产品含义、技术结构和生成结果能够持续保持一致的基础。

## 结果

共同语言会让 API 协作不那么脆弱。它减少翻译损耗，提升评审质量，也为契约优先设计提供了更稳定的基础。
