ApiHug Copilot 的设计目标，不是做一个独立聊天窗口，而是把 AI 能力嵌进 IDE 的真实工作流里。

## 它是什么

功能主要分布在三个位置：

- 用来输入 Prompt 和快捷指令的编辑器
- 右侧 Tool Window 或独立聊天面板
- 把当前代码上下文送入对话的编辑器操作入口

## 为什么重要

这样设计的好处是，查看代码、提问、复用 prompt 和处理结果可以连续发生，不需要在 IDE 和外部工具之间反复跳转。

## 它如何融入整体流程

输入编辑器在普通 Markdown 之上增加了几类 IDE 感知能力：

- 输入 `/` 可以打开 Prompt 自动补全
- 使用 `file:` 可以附加项目文件
- 如果没有匹配到快捷指令，也可以继续输入普通自然语言

### Tool Window 与独立面板

右侧 Tool Window 主要包含：

- 会话控制
- Vendor 与模型选择
- 主聊天区域
- 输入框和发送动作

如果右侧空间太挤，可以从 Tool Window 工具栏打开独立面板。两者能力一致，只是独立面板更宽。

### 对话面板

聊天面板支持：

- Markdown 渲染
- 项目文件链接
- Prompt 链接与上下文折叠
- 代码块复制、插入当前编辑器等操作
- 删除对话以重开一个干净线程

### 编辑器 Ask 入口
