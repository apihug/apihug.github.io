ApiHug Copilot 在 IntelliJ 里提供一个 AI 工作面，让你可以围绕代码、契约和实现上下文持续提问，而不必在多个工具之间来回切换。

## 它是什么

这一块能力主要由四部分组成：

- Vendor 与模型选择
- 可复用的 prompt 快捷指令
- 用于解释、评审和草拟代码的聊天面板
- 把编辑器上下文直接送进对话的操作入口

## 为什么重要

ApiHug 已经给项目建立了结构化契约模型，Copilot 的价值在于让这套结构真正进入日常开发流程：

- 理解已有代码和生成输出
- 标准化重复使用的 prompts
- 在项目上下文里做解释、评审和草稿生成
- 从设计问题顺滑过渡到实现问题

## 它如何融入整体流程

推荐这样使用：

1. 先在 [设置](/zhCN-docs/copilot/001_settings) 里配置 Vendor 和 Prompt
2. 再在 [功能说明](/zhCN-docs/copilot/002_functions) 里熟悉编辑器、聊天面板和右键入口
3. 当你需要遵循 ApiHug 规则和工作流时，再配合 [技能与规则](/zhCN-docs/skills) 一起使用
