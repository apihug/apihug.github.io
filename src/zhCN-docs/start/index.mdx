这一页带你从空目录走到一个可运行的 ApiHug 演示项目。路径很直接：先准备环境，再生成项目，接着跑通 `wire`、`stub` 和应用启动，最后检查生成出来的 OAS 接口文档。

<TipGood>
  ApiHug 3.x 面向 Spring Boot 4.0 及以上版本。升级老项目之前，建议先阅读 Spring Boot 4.0 迁移指南。
</TipGood>

## 你会完成什么

第一次体验通常会完成以下几件事：

1. 初始化一个 ApiHug 工作目录
2. 通过 IntelliJ 向导生成一个演示项目
3. 理解 `-proto` 模块与应用模块的分工
4. 依次执行 `wire`、`stub` 和应用启动
5. 打开生成出来的 OAS 文档

### 准备开发环境

先准备好基础环境：

1. JDK 17 或更高版本
2. Gradle 8 或更高版本
3. IntelliJ IDEA 2022 或更高版本
4. `ApiHug - API design Copilot` 插件
5. 可访问的 Spring Initializr，或你们团队自己的等价初始化流程

<DocsMavenCentralBadge />

### IntelliJ 插件

先从 JetBrains Marketplace 安装插件，然后重启 IntelliJ。

<DocsJetBrainsMarketplaceCard />

如果你后续更偏好命令行工作流，可以再配合 ApiHug REPL 使用。首次上手时，IntelliJ 向导仍然是理解项目结构最快的入口。

### 插件安装路径

1. 打开 `File > Settings > Plugins`
2. 搜索 `ApiHug`
3. 安装插件
4. 重启 IntelliJ

## 生成项目

IntelliJ 向导会直接给出一个适合 contract-first 开发的起点。

### 1. 创建 ApiHug 项目

1. 打开 `File > New > Project`
2. 选择 `ApiHug`
3. 进入项目设置步骤

<img src="/img/editor/002_start_project.gif" alt="从 ApiHug 插件创建项目" width={2316} height={1768} className="rounded-lg" />

### 2. 填写项目设置

设置 package、项目名、描述、SDK 选项、数据库厂商、缓存和端口。

<img src="/img/editor/003_project_settings.gif" alt="ApiHug 项目设置" width={2316} height={1768} className="rounded-lg" />

<TipGood>
  第一次使用时，除非团队已经有固定模板，否则建议先保留默认配置。
</TipGood>

一个典型的 ApiHug 项目通常从两个模块开始：

```text
order
|- order-app
`- order-app-proto
```

1. `order` 是根工作区
2. `order-app` 是手写业务逻辑所在的应用模块
3. `order-app-proto` 是驱动生成的契约模块

### 3. 选择 Spring 组件

这一段的心智模型和 [Spring Initializr](https://start.spring.io/) 一致：先选项目类型，再按需加入像 `Spring Web` 这样的 starter。

<img src="/img/editor/004_spring_settings.gif" alt="ApiHug 项目 Spring 设置" width={2316} height={1768} className="rounded-lg" />

完成后点击 **Create -> Open Project**。

### 4. 以 Gradle 方式加载项目

如果 IntelliJ 没有自动导入，请手动按 `Gradle` 项目重新加载。正常情况下你会看到 `demo-app` 和 `demo-app-proto` 两个模块。

<TipGood>
  升级已有项目时，先更新 `gradle/libs.versions.toml`，再重新加载 Gradle，确保 SDK 版本一致生效。
</TipGood>

<DocsMavenCentralBadge />

<img src="/img/editor/005_load_project.gif" alt="ApiHug 加载项目" width={2316} height={1768} className="rounded-lg" />

## 理解项目结构

最小示例可以参考 [simplest-demo](https://github.com/apihug/simplest-demo)：

```shell
+---gradle
|   |   libs.versions.toml  (1)
+---demo-app  (2)
|   |   build.gradle
|   \---src
|       \---main
|           +---java
|           |   \---com
|           |       \---apihug
|           |           \---demo
|           |               |   DemoAppApplication.java
|           |               +---domain
|           |               \---service
|           \---resources
|               |   hope-stub.json  (3)
|
+---demo-app-proto (4)
|   |   build.gradle
|   \---src
|       \---main
|           +---proto
|           |   \---com
|           |       \---apihug
|           |           \---demo
|           |               \---proto
|           |                   +---api
|           |                   \---infra
|           \---resources
|                   hope-wire.json (5)
```

| 文件                         | 作用                                 |
|------------------------------|--------------------------------------|
| **(1)** `libs.versions.toml` | 版本目录与插件别名配置               |
| **(2)** `demo-app`           | 应用模块与手写业务逻辑               |
| **(3)** `hope-stub.json`     | 应用模块的生成配置                   |
| **(4)** `demo-app-proto`     | protobuf 契约模块与 wire 输出        |
| **(5)** `hope-wire.json`     | proto 模块元数据                     |

## 跑通 Contract-First 工具链

### 1. 执行 Wire

`wire` 负责校验 proto 模块，并生成中立的契约构建输出。

1. 打开 `README.md > 0. Build All`
2. 复制命令到终端，例如 `gradlew clean build -x test -x wireTest -x stubTest`
3. 执行命令
4. 查看 `demo-app-proto` 下的生成结果

<img src="/img/editor/006_wire.gif" alt="ApiHug Wire 项目" width={2316} height={1768} className="rounded-lg" />

### 2. 执行 Stub

`stub` 会基于契约模块生成应用侧脚手架。

1. 打开 `README.md > 2. Build Stub (Individual)`
2. 执行命令，例如 `gradlew.bat demo-app:clean stub build -x test -x stubTest`
3. 查看 `demo-app` 下的生成结果

<img src="/img/editor/007_stub.gif" alt="ApiHug Stub 项目" width={2316} height={1768} className="rounded-lg" />

### 3. 启动应用

1. 打开 `README.md > 3. Run Application`
2. 执行 `gradlew demo-app:bootRun`
3. 确认应用正常启动

<img src="/img/editor/008_boot.gif" alt="ApiHug Spring Boot" width={2316} height={1768} className="rounded-lg" />

```shell
Application 'demo-app' is running! Access URLs:

Local                             http://localhost:18089/
OAS                               http://localhost:18089/v3/api-docs
Actuator                          http://localhost:18089/management
Api-Errors                        http://localhost:18089/hope/meta/errors
Api-Dictionaries                  http://localhost:18089/hope/meta/dictionaries
Api-Authorities                   http://localhost:18089/hope/meta/authorities
Profile(s)                        dev
```

### 4. 检查 OAS 输出

把控制台里的 `/v3/api-docs` 地址复制到浏览器，确认生成出来的接口面没有问题。

<img src="/img/editor/009_OAS.gif" alt="ApiHug API 信息" width={2316} height={1768} className="rounded-lg" />
