## 它是什么

UI 层并没有引入另一套独立 DSL。它仍然基于 ApiHug 使用的 ProtoBuffer 契约与扩展规则，只是在前端侧消费这些元数据。

## 适用场景

当你想确认“前端到底依赖哪一套规范”时，就应该看这一页。

## 关键结论

1. ApiHug UI 不会绕开 Proto 契约。
2. 前端生成依赖的模型、路由、表单和接口描述，都来自同一份协议定义。
3. 这意味着前后端仍然共享同一个 source of truth。
