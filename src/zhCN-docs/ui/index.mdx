## 它是什么

UI 章节说明 ApiHug 如何把契约优先设计延伸到前端层，包括 proto 对前端的约束、Vue 应用组织方式，以及生成式 UI 工具链。

## 适用场景

当你希望：

- 基于 ApiHug 契约组织前端项目
- 让路由、服务层、i18n 与生成结果保持一致
- 减少前后端对齐时的重复手工工作

就应该从这一节开始。

## 推荐阅读顺序

1. [协议层](/zhCN-docs/ui/001_proto)
2. [Vue 框架](/zhCN-docs/ui/002_vue)
3. [应用配置](/zhCN-docs/ui/003_app)
4. [工具链](/zhCN-docs/ui/004_tool_chain)

## 结果

读完这一节后，你应该能理解 ApiHug UI 不是“独立前端模板”，而是和后端契约、生成数据层、应用配置一起协作的完整前端工作流。
