## 它是什么

UI 章节说明 ApiHug 如何把契约优先设计扩展到前端层。内容包括面向 proto 的模型约束、基于 Vue 的应用组织方式，以及把生成元数据转成可用 UI 代码的工具链。

## 适用场景

当你希望：

- 基于 ApiHug 契约生成或组织前端项目
- 理解 UI 模块如何与后端项目结构配合
- 把路由、服务、i18n 与生成结果放进同一条工作流

就应该先阅读这一节。

## 推荐阅读顺序

1. [协议层](/zhCN-docs/ui/001_proto)
2. [Vue 框架](/zhCN-docs/ui/002_vue)
3. [应用配置](/zhCN-docs/ui/003_app)

## 结果

读完这一节后，你应该能理解 ApiHug UI 并不是“独立前端模板”，而是把前端结构、生成产物和后端契约持续对齐的一套完整方案。
