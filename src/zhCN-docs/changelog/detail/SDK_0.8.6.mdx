## 问题

ApiHug 一直希望生成代码能够稳定到可以直接放进版本库里。

在 `0.8.6-RELEASE` 之前，`stub` 与 `wire` 的部分生成流程仍然依赖 `HashMap` 的遍历顺序。即使 schema 或模型只发生了很小的变化，也可能让生成结果出现很多无意义的 diff。

## 这次改动解决了什么

从 `0.8.6-RELEASE` 开始，生成文件会以更稳定的顺序输出，因此升级后的 diff 更容易阅读，也更容易合并。

这一版也延续了 [SDK 0.7.8](/zhCN-docs/changelog/detail/SDK_0.7.8) 中对 repository 维护方式的重构。

Latest: [Maven Central](https://search.maven.org/artifact/com.apihug/it-bom)
