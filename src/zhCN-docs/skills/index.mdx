# 技能与规则

这组文档只覆盖 ApiHug 自己维护的 AI 工作流表面：规则、contract-first 交付流程，以及在这些边界内工作的核心技能。

## 它是什么

这里主要覆盖三类内容：

- 用来约束设计与实现的框架规则
- 从 proto 到交付的 contract-first 工作流
- 创建、实现、评审 story 时会用到的核心技能

它不是整个 BMAD 目录的镜像，而是只保留那些真正影响 ApiHug 项目落地的部分。

## 为什么重要

AI 工作流只有在边界清晰时才可靠。ApiHug Skills 解决的是“所有人到底按什么模型协作”这个问题：

- proto 是 source of truth
- 生成物继续保持可生成
- 手写代码只放在允许手写的层
- 评审依据是 ApiHug 规则，而不是泛化的建议

## 它如何融入整体流程

ApiHug 借用了 BMAD 的规划与执行思路，但把它收束到了 ApiHug 栈本身：

- protobuf-first 的 API 与领域设计
- 明确的后端与前端生成边界
- 由规则驱动的契约评审和实现评审

也正因为如此，工作流页面和规则页面必须配套阅读。更大的 **ApiHug Skills** 仓库里可能还有其他内容，但站内文档只保留与 ApiHug 项目直接相关的部分。
