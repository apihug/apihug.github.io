ApiHug revolutionizes your API lifecycle with innovative ideas and thoughtful features.

Streamline the entire API journey — from Design, Build, Test to Integrate — making it faster and easier than ever before.

Unite all roles on a single platform, enabling seamless collaboration throughout the entire lifecycle of APIs.

## Why should I use ApiHug?

ApiHug is user-friendly, design-driven, straightforward and satisfying.

ApiHug revolutionizes the separation of API design and implementation by adopting a domain-specific language (DSL) built on Protocol Buffers, employing object-oriented methods to design and define API components. This approach significantly enhances API reusability.

ApiHug leverages Git for API management and utilizes repositories for API distribution. By building upon existing systems and frameworks, ApiHug completely transforms the paradigm of API development and management.

### Key features

- Effortless visual API design with IDEA plugin integration
- Object-oriented (domain-specific language) approach for highly reusable API design
- Compliance with OAS design standards for industry best practices
- Design, test and implementation on one platform
- Integration with existing CI/CD facilities
- Built on mature toolchains (Gradle), IDEs (IntelliJ IDEA), and frameworks (Spring Boot)

## What is ApiHug used for?

### API Design

ApiHug is a design-first platform that follows the standard [OAS (3.1.0)](https://spec.openapis.org/oas/latest.html). It introduces a Common Language (DSL) to bridge the gap between:

- Domain Experts
- Technical Experts
- Working Software and Tests
- Third-party Partners

With its intuitive interface, ApiHug empowers the complete API design lifecycle.

### API Implementation

Accelerate your development by converting your APIs into Software Artifacts:

- API Definitions Meta
- Application Framework Backend (Spring Boot / Spring Cloud)
- Tool Chain: Gradle + IDEA Plugin

### API Management

ApiHug revolutionizes API management by bringing common project management concepts to the full API lifecycle. With native Git support, it simplifies version control and enables users to seamlessly design, document, mock, test, and share APIs.

### API Distribution

Distributing different versions of APIs has always been a challenge. ApiHug tackles this by leveraging existing package distribution mechanisms, such as Maven repositories, to ensure a more robust and controllable API distribution process.

### API Integration

API design and implementation typically focus on solutions for API providers. However, APIs need to be consumed by API consumers.

ApiHug offers a solution for API consumer clients. The API distribution mechanism ensures synchronization of different API versions, and API integration enables automatic generation of client-side stub code.

## FAQ

### Is ApiHug free?

Absolutely! ApiHug can be utilized in various types of projects, including commercial ones. It offers versatility and flexibility to cater to your project requirements.

### Is ApiHug open source?

The SDK is available from the public Maven repository. The IDEA plugin source code is not shared.
