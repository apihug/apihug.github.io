Principles

ApiHug 准则

单一信任源

开发者友好，无缝协同，原生Git的协作，用于全周期 API 开发，简化 API 管理、团队协作和共享。

共同语言

领域特定语言（DSL）：弥合领域专家和技术专家之间的鸿沟，在整个 API 生命周期中消除语言和技术障碍。

集成开放

基于现有的工具、框架和标准，最大化利用组织资产，降低迁移和学习成本，无缝集成到其他平台。

面向全角色平台

设计，测试，实现，管理，不同团队角色

  • 可视化API设计，与 IDEA 无缝集成
  • 领域+面向对象设计，API设计高复用
  • 符合OAS，遵循行业最佳实践
